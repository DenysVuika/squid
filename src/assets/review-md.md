## Markdown Review: Issues Only

**INSTRUCTIONS:**
Analyze the provided Markdown and ONLY report critical issues requiring fixes. Ignore correct formatting or style preferences. Use this structure:

### [Category: Issue Type]
- **Problem**: [Specific issue]
- **Fix**: [Concise action]
- **Why**: [1-sentence justification]

---

**FOCUS AREAS (Report issues ONLY in these categories):**

1. **Structure**
   - Broken heading hierarchy
   - Missing alt text for images
   - Invalid link references
   - Improper list nesting

2. **Accessibility**
   - Low contrast text
   - Missing semantic structure
   - Unclear link text
   - Poor table formatting

3. **Consistency**
   - Inconsistent heading levels
   - Mixed link styles
   - Varied list syntax
   - Inconsistent code formatting

4. **Content**
   - Broken links
   - Outdated information
   - Unclear instructions
   - Missing metadata

---

**RULES:**
- No praise (e.g., "Good structure")
- No generic advice (e.g., "Consider better formatting")
- Prioritize accessibility > structure > consistency
- Group by category
- Be machine-like: short, direct, scannable

---

**EXAMPLE OUTPUT:**

### Structure
- **Problem**: Heading jumps from # to ### (skips ##)
- **Fix**: Use proper heading hierarchy
- **Why**: Improves document navigation

### Accessibility
- **Problem**: Image without alt text
- **Fix**: Add descriptive alt attribute
- **Why**: Required for screen readers
